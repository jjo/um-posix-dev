Estructuración del proyecto

  • Fuentes
    • Interface: .h
    • Implementación: .c -> .o
  • Entorno de trabajo
    • Proyecto: Makefile
      • Relación 
        	objectivo(s): fuentes ...
		commandos ...
      • Objetivos típicos:
        • all: crea todos los "targets"
        • clean: limpia "targets", objetos
        • test: prueba la aplicacion
        • dist: crea .tar
      • Variables comunes:
        • CFLAGS: parámetros para el compilador
        • LDFLAGS: parám. para el linker
        • LDLIBS: libs. para el linker
      • Operación make/vim
    • Un poco más de vim ...

      Uso de :make, ^Z, :sp {arch}

  • Debugger: gdb
    • Intro: señales, core files.
    • Invocación, uso runtime y postmortem (core)
    • Comandos más comunes:
      • run
      • break {funcion|linea}
      • cont, step
      • bt (backtrace)
      • print variable
      • x dir_memoria
    • Ciclo edición, compilación y debugging

    Librería: manejo de opciones de comandos
